## Redis 数据同步工具设计文档

### 1. 概述
本工具用于在不同 Redis 实例间进行数据同步，支持**断点续传**。  
通过模拟 Redis 主从复制协议，工具可从源端 Redis 拉取全量或增量数据，并写入目标 Redis。  
同步过程借助自定义的 Checkpoint 机制记录进度，避免重启后重复拉取。

### 2. 核心流程

整个同步流程分为 **启动恢复** → **建立复制** → **数据传输** 三个阶段。

```
┌─────────────────────────────────────────────────────┐
│ 1. INFO REPLICATION 获取源端 master_replid         │
└──────────────┬──────────────────────────────────────┘
               ▼
┌─────────────────────────────────────────────────────┐
│ 2. HGET redis_ha_tool_checkpoint_hash $master_replid│
│    获取当前生效的 Checkpoint 名称                   │
└──────────────┬──────────────────────────────────────┘
               ▼
┌─────────────────────────────────────────────────────┐
│ 3. HGET redis_ha_tool_checkpoint $checkpoint_name   │
│    得到 Checkpoint 详情（包含 offset）              │
└──────────────┬──────────────────────────────────────┘
               ▼
┌─────────────────────────────────────────────────────┐
│ 4. PSYNC $master_replid $offset                     │
│    向源 Redis 请求从该 offset 开始复制              │
└──────────────┬──────────────────────────────────────┘
               ▼
        ┌──────┴──────┐
        │  源端回复    │
        └──────┬──────┘
   FULLRESYNC│          │ CONTINUE
             ▼          ▼
    ┌────────────┐  ┌────────────┐
    │ 全量同步    │  │ 增量同步    │
    │ (RDB传输)  │  │ (命令流)   │
    └────────────┘  └────────────┘
```

#### 详细步骤说明

**步骤1：获取 Replication ID**  
连接源端 Redis，执行 `INFO REPLICATION`。若源端为 Master，则可获得 `master_replid`；若源端为 Slave，需根据场景确认是否使用其 `master_replid`（建议工具直连 Master）。  
此 ID 用于唯一标识数据集的历史分支。

**步骤2：查询当前 Checkpoint 名称**  
在工具专用的管理 Redis（或本地存储）中执行：  
`HGET redis_ha_tool_checkpoint_hash <master_replid>`  
返回当前使用的 Checkpoint 名称（例如 `e34950d6e4a6ede74bf5717d25648898a5d71e05_checkpoint`）。若无记录，说明是首次同步，直接进入全量流程。

**步骤3：获取 Checkpoint 详情**  
执行：  
`HGET redis_ha_tool_checkpoint <master_replid>_offset`  
返回当前checkpoint的offset

**步骤4：发起部分同步请求**  
向源 Redis 发送 `PSYNC <master_replid> <offset>`。  
- 若 `replid` 相同且 `offset` 在复制积压缓冲区内，源端返回 `+CONTINUE`，随后发送增量命令流。  
- 否则返回 `+FULLRESYNC <new_replid> <new_offset>`，需要全量同步。

**步骤5：分支处理**  
- **FULLRESYNC**：进入 RDB 全量复制流程。  
- **CONTINUE**：直接使用原连接进行增量命令同步。

**步骤6：RDB 全量同步**  
源 Redis 将 RDB 文件通过网络发送给工具。工具将 RDB 数据先完整写入**本地磁盘缓存**，然后在目标 Redis 上执行 `RESTORE` 或直接解析 RDB 流逐个写入 Key。  
本地缓存可提高可靠性，防止网络中断导致目标 Redis 数据不全。

**步骤7：增量命令同步**  
在 `CONTINUE` 或全量 RDB 传输完成后，源端会持续发送 AOF 格式的命令流（以 Redis 复制协议编码）。工具需解析这些命令，将其转换为目标 Redis 可执行的写命令，如 `SET`、`DEL`、`HSET` 等，逐条发送到目标端。  
同时工具应周期性更新 Checkpoint 中的 `offset`，记录已应用的命令偏移量。

### 3. Checkpoint 设计

Checkpoint 数据存储在两个 Hash 结构中（可位于独立 Redis 或工具内置存储）：

- `redis_ha_tool_checkpoint_hash`  
  - 键：`master_replid`  
  - 值：当前激活的 Checkpoint 名称（字符串）

- `redis_ha_tool_checkpoint`  
  - 键1：`master_replid`_offset, 值1：已同步的复制偏移量
  - 键2：`master_replid`_mtime, 值2: 最后更新时间  
  - 键3：`master_replid`_runid, 值3: 当时对应的 master_replid  
  - 键4：`master_replid`_version, 值4: 版本号

**更新时机**：每当从源端收到一个命令并成功写入目标后，可积累一定数量或按时间更新 创建checkpoint。  
**重置逻辑**：全量同步完成后，需用创建 Checkpoint，并更新 `redis_ha_tool_checkpoint_hash` 的指向。
