# Redis-HA-Tool
一站式企业级 Redis 数据治理工具，面向缓存集群迁移、异地灾备、双活多活、数据备份恢复全场景，基于 Redis Replication 协议实现全量+增量数据流转，内置高可用故障切换与断点续传能力，解决传统同步工具无HA、中断重跑、仅单向同步、备份恢复割裂等痛点。

## 一、项目简介
`Redis-HA-Tool` 是面向生产环境打造的一体化 Redis 数据运维套件，统一覆盖**离线备份、数据恢复、跨集群单向复制、双向双活同步**四大核心能力。
兼容 Standalone、Sentinel、Redis Cluster 多架构，适用于机房迁移、异地容灾、同城多活、故障回滚、数据冷热归档等业务场景，具备企业级稳定性与链路自愈能力。

### 核心关键词
Redis同步、Redis双活、跨集群迁移、断点续传、主备高可用、故障自动切换、RDB备份、数据恢复、单向复制、双向同步、增量同步、全量同步、异地多活、缓存灾备、Replication协议、链路自愈

## 二、核心能力一览
### 1. 数据备份 & 恢复（离线数据治理）
- **Dump 全量备份**：解析源库 RDB/AOF，导出标准化备份文件，支持定时自动备份、分片集群并行导出
- **Restore 数据回滚**：本地备份文件一键恢复至任意 Redis 实例/集群，支持分库、Key 前缀过滤导入
- 适配数据归档、故障应急回滚、测试环境快速造数场景

### 2. 单向数据同步（迁移/灾备）
模拟 Redis Slave 拉取 Replication 数据流，全量初始化+实时增量同步：
- 支持单机 ↔ 哨兵 ↔ Cluster 异构集群互通
- 业务不停服迁移、异地只读灾备、数据分流镜像场景专用

### 3. 双向数据同步（同城/异地多活）
内置循环写入防冲突机制，支持双集群实时双向数据流转：
- 双写多活架构、同城双机房高可用、异地单元化业务
- 自动识别同源同步指令，避免环形复制数据风暴

### 4. HA 高可用架构（工具自身高可用）
- 同步服务主备双节点部署，故障自动 Failover
- 主节点宕机后备节点自动接管同步链路，不中断数据流转
- 任务元数据持久化，切换后无缝承接增量同步

### 5. 断点续传（链路自愈核心特性）
- 持久化同步位点（repl offset / AOF 偏移）
- 网络抖动、进程重启、节点宕机后，从上次中断位置增量续传，无需重跑全量
- 支持长断连场景，自动追平积压增量数据，大幅降低重同步耗时

## 三、产品核心特性

✅ **全链路一体化**：备份、恢复、单向同步、双向双活同一工具闭环，无需多组件组合

✅ **原生断点续传**：持久化同步位点，中断免全量重传，大幅节省大集群迁移成本

✅ **工具层 HA 主备切换**：同步服务集群化部署，故障自动切换，保障7×24小时同步稳定

✅ **双模式复制支持**
  - 单向同步：集群迁移、异地只读灾备、数据镜像
  - 双向同步：同城多活、异地双机房单元化业务

✅ **全架构兼容**：Standalone / Sentinel / Redis Cluster 互通同步

✅ **低侵入同步**：基于 Redis 原生 Replication 协议，源库无需改造、无业务侵入

✅ **全量+增量双通道**：首次全量初始化，后续实时增量追数，支持不停机迁移

✅ **数据一致性保障**：双向同步内置防循环复制策略，避免数据冲突、重复写入

✅ **轻量化部署**：单二进制包运行，支持 Docker、K8s 容器化运维

# Redis 全系列高可用架构对比 \+ 自研多集群双活增强架构（官方推荐终版架构）

## 一、原生 Redis 官方所有架构汇总（优缺点 \& 适用场景）

为了清晰体现本项目的架构价值，我们先完整梳理业界所有标准 Redis 多机 HA 架构，并明确其**原生天生缺陷**，最后引出本项目自研的企业级终极架构。

### 1\.1 纯主从架构（基础架构，无HA）

**架构拓扑**：1Master \+ N Slave

**能力**：数据单向复制、读写分离、数据备份

**致命缺陷**：无自动故障转移、无监控、无法跨集群同步、不支持异地容灾

**定位**：非高可用架构，生产禁止独立使用

### 1\.2 Sentinel 哨兵架构（单集群HA，中小业务标准）

**架构拓扑**：1主多从 \+ 3/5哨兵节点监控

**能力**：单集群内自动故障转移、节点高可用、数据多副本

**原生短板**：

- **不支持数据分片**，单节点容量瓶颈严重

- **仅支持集群内单向复制**，无法多集群互通

- **无异地多活能力**，只能单集群HA，机房级故障无法兜底

- 无官方备份恢复、断点续传能力

**定位**：单机房、小数据量、无异地容灾需求业务

### 1\.3 Redis Cluster 原生分片集群（分布式HA，业界主流）

**架构拓扑**：3主3从及以上分片架构，每分片自带主从HA

**能力**：数据分片、水平扩容、单集群内故障自动切换、海量数据存储

**原生致命短板（业界长期痛点）**：

- **绝对无法跨集群同步**：多套独立 Cluster 集群完全孤岛，原生不支持集群间数据同步

- **仅单向主从复制**，完全**不支持双向同步、异地双活**

- 无断点续传，集群迁移/同步中断必须全量重传

- 无统一备份恢复体系，数据误删、集群损坏难以回滚

- 多机房部署只能做节点交叉部署，**无法实现真正异地多活互写**

**定位**：单机房分布式缓存，**不具备企业级异地容灾、双活能力**

### 1\.4 Proxy 代理集群（Codis/Redis\-Proxy）

**架构拓扑**：Proxy 接入层 \+ 后端 Cluster 集群

**能力**：屏蔽分片、统一接入、限流路由

**短板**：仅解决接入层问题，**完全没有解决多集群数据互通、双向多活、灾备同步问题**

## 二、原生所有架构的共性瓶颈（行业痛点）

无论哨兵、原生 Cluster、Proxy 架构，官方原生能力存在无法突破的天花板：

- ✅ 只能**单集群内部HA**，**多集群数据完全孤岛**

- ✅ 只支持**单向主从复制**，无任何官方双向同步、双活方案

- ✅ 无集群级备份恢复、无断点续传同步机制

- ✅ **无法实现真正异地双活、跨机房容灾**，只能做简单冷备

## 三、本项目：Redis 多集群全域双活高可用架构

**本架构为原生 Redis 架构的超集与终极增强**，兼容：单机 / Sentinel / Redis Cluster 所有底层架构，同时补齐官方所有短板，是目前唯一支持**集群级双向多活、跨机房实时同步、全域数据容灾**的一体化架构。

### 3\.1 整体架构拓扑（标准生产架构）

**多集群分层高可用架构**

```Plain Text
# 异地机房A（主业务集群）
Redis Cluster / Sentinel 主集群（提供读写服务）
        ↕️（本工具双向实时同步+断点续传）
# 异地机房B/备用集群
Redis Cluster / Sentinel 备用集群（双活互写/灾备兜底）

# 本系统核心能力层（贯穿所有集群）
1. 集群级全量/增量数据备份 & 定点恢复
2. 跨集群 单向同步 / 双向双活同步
3. 同步链路断点续传、自动续跑
4. 跟随Redis主备切换，链路无感HA
5. 多集群数据一致性校验、冲突解决
```

### 3\.2 架构核心能力（独有、原生不具备）

#### ✅ 1\. 真正的**集群级跨机房双向双活**

打破原生单集群孤岛限制，支持**多套独立 Redis Cluster / 哨兵集群**之间双向实时数据同步，多机房业务可同时读写，是真正意义上的企业级异地多活架构，而非原生伪双活。

#### ✅ 2\. 灵活的多集群单向/双向同步能力

支持自由切换同步模式：

- **单向同步**：生产→灾备、线上→测试、数据镜像下沉

- **双向同步**：同城双活、异地多活、双集群平滑迁移

#### ✅ 3\. 工业级断点续传同步机制

全量同步、增量同步均支持断点续传，网络抖动、服务重启、集群节点变更后，自动从历史位点继续同步，无需全量重传，解决TB级大集群同步耗时、带宽压力大的行业难题。

#### ✅ 4\. 全域数据备份 \& 定点恢复体系

支持单机、哨兵、Cluster 全架构集群级备份，支持时间点恢复、跨集群恢复、误删数据回滚，补齐 Redis 无官方统一数据恢复能力的短板。

#### ✅ 5\. 同步链路自带高可用（跟随Redis主备切换）

自动感知 Redis 集群主备切换、哨兵故障转移，同步链路自动适配新主节点，无需人工修改配置；同步服务本身支持主备部署，彻底消除同步链路单点故障。
